# MissionManager

**大事な締め切りを見逃さない。シンプルで、迷わない。**

---

MissionManager は、プロジェクト単位でタスクを整理し、期限に集中して取り組めるデスクトップアプリです。  
ジャンル → ミッション → タスクという直感的な3階層で、やるべきことをスッキリ管理できます。

![PySide6](https://img.shields.io/badge/GUI-PySide6-41CD52?style=flat&logo=qt&logoColor=white)
![License](https://img.shields.io/badge/License-MIT-green?style=flat)

## なぜ MissionManager か？

| よくある課題 | MissionManager のアプローチ |
|-------------|-----------------------------|
| タスクが散らばって優先度が曖昧 | **期限が近く、未完了のものを自動で上に表示** — 次にやるべきことが一目でわかる |
| 大きな目標が把握しづらい | **ジャンル・ミッションでグルーピング** — プロジェクト単位で進捗を整理 |
| 進捗が見えない | **プログレスバーで進捗を可視化** — やった分が一目で伝わる |
| 手動での並び替えが面倒 | **右クリックメニューで直感的に操作** — ドラッグ不要でスムーズ |

## 主な機能

### 3階層構造で整理

```
ジャンル（例: 仕事 / 勉強 / 個人）
  └── ミッション（例: 〇〇プロジェクト）
        └── タスク（具体的なTODO）
```

プロジェクトやテーマごとにミッションを分け、その中にタスクを並べるだけで、自然と「何を」「いつまでに」が整理されます。

### 期限が近いものから自動で並ぶ

ミッション・タスクともに、**期限が近く未完了のものが自動で上に表示**されるため、  
毎回「何から手をつけようか」と考えなくて済みます。

### シンプルな操作

- 右クリックメニュー中心の操作
- クリックでタスクの表示／非表示を切り替え
- 追加・編集・削除・順序変更を一画面で完結
- 変更は自動保存 — 保存ボタン不要

### ローカル完結

- データはあなたのPC内のJSONファイルに保存
- クラウド同期不要、オフラインで使える

## クイックスタート

### 1. インストール

```bash
git clone https://github.com/yourusername/MissionManager.git
cd MissionManager
pip install -r requirements.txt
```

### 2. 起動

```bash
python main.py
```

### 3. 最初の一歩

1. 上部の「追加」でジャンルを作成（例: 「仕事」）
2. 下部の「ミッション追加」でミッションを作成（例: 「新機能開発」）
3. ミッションカードをクリックして展開 → 「タスク追加」でタスクを登録
4. 右クリックで期限を設定 — 期限が近いものから自動で上に表示されます

---

## 操作ガイド

### ジャンル

| 操作 | 方法 |
|------|------|
| 追加 | 上部の「追加」ボタン |
| 名前変更・概要編集・削除・順序変更 | ジャンル選択コンボボックスを右クリック |

### ミッション

| 操作 | 方法 |
|------|------|
| 追加 | 下部の「ミッション追加」ボタン |
| 名前変更・概要編集・期限設定・削除・順序変更 | ミッションカードを右クリック |
| タスクの表示切替 | ミッションカードをクリック |

### タスク

| 操作 | 方法 |
|------|------|
| 追加 | ミッション展開後、「タスク追加」ボタン |
| 完了/未完了 | チェックボックスをクリック |
| 名前変更・期限編集・削除・順序変更 | タスクを右クリック |

## 技術スタック

- **Python 3.10+** — 型ヒントを活用した保守しやすいコード
- **PySide6** — モダンなデスクトップGUI
- **TypedDict** — 型安全なデータ構造
- **クリーンアーキテクチャ** — UI層・ビジネスロジック層・データ層の分離
- **依存性注入（DI）** — 疎結合で拡張しやすい設計

## プロジェクト構造

```
MissionManager/
├── main.py                    # エントリーポイント
├── missionmanager/
│   ├── models.py              # データモデル・ソートロジック
│   ├── storage.py             # JSON永続化
│   ├── app.py                 # ビジネスロジック（AppService）
│   └── ui/
│       ├── views.py           # メインウィンドウ
│       ├── mission_card.py    # ミッションカード
│       ├── task_item.py       # タスクアイテム
│       └── date_dialog.py     # 期限入力ダイアログ
├── data/                      # データ保存（自動生成）
└── requirements.txt
```

## データ形式

データは `data/app_data.json` に JSON 形式で保存されます：

```json
{
  "genres": [
    {
      "name": "仕事",
      "missions": [
        {
          "name": "新機能開発",
          "tasks": [
            { "name": "設計書作成", "done": false, "due_date": "2025-02-25" }
          ],
          "due_date": "2025-03-01",
          "completed_at": null
        }
      ]
    }
  ]
}
```
